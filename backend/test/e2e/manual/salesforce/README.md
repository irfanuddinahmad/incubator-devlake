# Salesforce Manual E2E

These tests connect to a real Salesforce org and are meant to run manually.
Do not commit real Salesforce tokens, client secrets, org IDs, or screenshots.

## Quick Checks

Run these from `backend` before the live e2e test:

```bash
go test ./plugins/salesforce/...
go test ./test/e2e/manual/salesforce/oauth_helper -count=1
go test ./test/e2e/manual/salesforce -run TestDoesNotExist -count=1
```

## Generate Local OAuth Config

Create a Salesforce External Client App or Connected App with:

- Callback URL: `http://localhost:1717/callback`
- OAuth scopes: `api` and `refresh_token` / `offline_access`

Then run the helper from `backend`:

```bash
go run ./test/e2e/manual/salesforce/oauth_helper \
  -client-id "YOUR_CONSUMER_KEY" \
  -client-secret "YOUR_CONSUMER_SECRET" \
  -login-url "https://login.salesforce.com" \
  -occurred-after "2026-04-01T00:00:00Z"
```

For sandbox orgs, use:

```bash
-login-url "https://test.salesforce.com"
```

For Developer Edition / My Domain orgs, use your org login domain:

```bash
-login-url "https://your-domain.my.salesforce.com"
```

The login URL must use `https://`.

The helper opens Salesforce in your browser, captures the callback code on localhost, exchanges it for tokens, and writes:

```text
backend/test/e2e/manual/salesforce/salesforce_local_test.go
```

That file is ignored by git through `backend/test/e2e/manual/.gitignore`.
If the file already exists, rerun with `-force=true`.

## Prepare Test Database

The live e2e test drops/recreates tables in the configured database. Use a disposable database, not the normal `lake` database.
The test refuses to run unless `E2E_DB_URL` points to a test-looking database name such as `lake_test`.

If you use `docker-compose-dev.yml`, create and grant the test DB once:

```bash
docker compose -f docker-compose-dev.yml exec -T mysql mysql -uroot -padmin \
  -e "CREATE DATABASE IF NOT EXISTS lake_test; GRANT ALL PRIVILEGES ON lake_test.* TO 'merico'@'%'; FLUSH PRIVILEGES;"
```

## Run The Test

Run the live test from `backend`:

```bash
E2E_DB_URL="mysql://merico:merico@127.0.0.1:3306/lake_test?parseTime=true" \
  go test ./test/e2e/manual/salesforce -run TestSalesforcePlugin -count=1 -v
```

If you intentionally need to run against another database, set `SALESFORCE_E2E_ALLOW_UNSAFE_DB=true`.
Do not use that override against a real DevLake database.

The test verifies:

- Salesforce connection creation and test-connection flow
- Salesforce org remote scope discovery
- Scope config and scope creation
- Blueprint creation and pipeline execution
- `user_activities` rows for `source_system = 'salesforce'`

## Notes

- `parseTime=true` is required so MySQL timestamp columns scan into Go `time.Time`.
- The test sets `DISABLED_REMOTE_PLUGINS=true` to avoid requiring local Python/Poetry remote plugin dependencies.
- The generated `salesforce_local_test.go` should stay local and ignored.
- Runtime logs under `backend/test/e2e/manual/salesforce/logs` are generated by the local test server and should not be committed.
